# Task: Create 3 test cron jobs via OpenClaw CLI

## Description

Create 3 cron jobs with different schedules to validate OpenClaw's scheduling system. Use `openclaw cron add` CLI or the GUI Cron page.

## Acceptance Criteria
- [ ] Job 1: Heartbeat check every 5 minutes — simple echo/status
- [ ] Job 2: Hourly system status — report gateway health
- [ ] Job 3: Daily summary at 9 AM (UTC+8 = 1 AM UTC) — generate report
- [ ] All 3 jobs visible in GUI dashboard (定時任務: 3 jobs)
- [ ] At least Job 1 has execution history visible

## Technical Details

Run via kubectl exec into the gateway pod:

```bash
# Job 1: Heartbeat every 5 minutes
openclaw cron add --name "heartbeat" --schedule "*/5 * * * *" \
  --prompt "Report current system status in one sentence"

# Job 2: Hourly status
openclaw cron add --name "hourly-status" --schedule "0 * * * *" \
  --prompt "Summarize gateway health: uptime, active channels, recent errors"

# Job 3: Daily summary at 1:00 UTC (9:00 AM UTC+8)
openclaw cron add --name "daily-summary" --schedule "0 1 * * *" \
  --prompt "Generate a daily summary: messages processed, costs, channel status, any issues detected"
```
